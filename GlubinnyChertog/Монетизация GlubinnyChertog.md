# Монетизация: сети, медиация и выплаты

## Проблема

AdMob (Google) не выплачивает на российские банковские реквизиты — известное ограничение с 2022 года. Это блокирует стандартную связку "AdMob + rewarded/interstitial", если разработчик и его счета находятся в РФ.

---

## Альтернативные рекламные сети и платформы

| Платформа | Тип | Комментарий |
|---|---|---|
| **Unity Ads** | Рекламная сеть | Выплаты доступны, стандартные форматы (rewarded video, interstitial), неплохой eCPM в глобальном миксе |
| **Bigo Ads** | Рекламная сеть | Альтернатива для rewarded video; нужно проверять актуальные условия выплат на момент интеграции — рынок меняется быстро |
| **CAS.AI** | Медиатор (объединяет несколько сетей через одну SDK) | Отзывы смешанные — стоит протестировать на небольшом объёме трафика и сравнить реальный eCPM/fill rate с альтернативами, а не доверять только заявленным цифрам |
| **X3M (медиация) + биллинг Яндекса** | Медиатор с российским биллингом | Закрывает именно проблему вывода денег для рос. разработчиков — выплаты идут через Яндекс. Нужно уточнить актуальные комиссии и минимальный порог выплат перед интеграцией |
| **Яндекс Мобильная медиация / РСЯ** | Медиатор + собственная сеть | Бесплатная медиация от Яндекса, объединяет несколько сетей (включая AdMob, MyTarget, AppLovin) через единый кабинет; есть вариант "Простая монетизация" — выплаты единым платежом, минимальный порог 5 000 ₽, комиссия 10% от рекламного вознаграждения. Выплаты ежемесячно при достижении порога |

## Что важно проверить до финального выбора

- [ ] Актуальные условия выплат и комиссии каждой платформы (могут меняться чаще раза в год — проверять перед интеграцией, а не доверять старым данным)
- [ ] Поддержка обоих нужных форматов — rewarded video + IAP — каждой платформой/медиатором
- [ ] Реальный eCPM/fill rate на тестовом трафике, а не только маркетинговые обещания сети
- [ ] Минимальный порог вывода и периодичность выплат (еженедельно / ежемесячно)
- [ ] Совместимость SDK с версией проекта (Unity 6000.3.18f1) — проверить актуальность адаптеров медиации под эту версию Unity
- [ ] Если таргет глобальный (не только РФ) — нужна ли отдельная схема монетизации/выплат для не-российского трафика, или одна медиация покроет всё

---

## Рекомендация по архитектуре монетизации

Не привязываться к одной сети. Использовать медиацию (например, X3M или Яндекс Мобильная медиация) с несколькими подключёнными сетями в waterfall/in-app bidding — это:
- снижает зависимость от условий одной платформы,
- повышает заполняемость рекламных блоков (fill rate),
- упрощает получение выплат единым платежом вместо нескольких разных кабинетов.

При глобальном таргетинге стоит протестировать, насколько хорошо российские медиаторы (X3M, Яндекс) работают с не-российским трафиком — возможно, потребуется гибридная схема: Яндекс/X3M для аудитории СНГ, Unity Ads/Bigo для остального мира.

## Открытые вопросы для уточнения позже

- Точные текущие комиссии CAS.AI и X3M на момент интеграции (рынок медиаторов быстро меняется)
- Реальные отзывы от разработчиков, использующих X3M именно для казуальных/гиперказуальных игр (а не только гиперкэжуал-паблишеров)
- Минимальный трафик/доход, при котором переход на платный медиатор (CAS.AI) оправдан по сравнению с бесплатной Яндекс-медиацией
